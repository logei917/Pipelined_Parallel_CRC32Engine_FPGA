# 256-bit Hardware-Accelerated CRC-32C Verification Engine

This repository contains the source code, testbenches, MATLAB generation scripts, and the Python golden model for the master's thesis project **"A 256-bit Hardware-Accelerated CRC-32C Verification Engine for High-Frequency Trading on HKEX"**. The design progresses through three architectural stages and is implemented on a Xilinx Artix-7 FPGA using Vivado.

## Folder & File Description

### Root
- **CRC32_Verification.py** – Python reference model that computes CRC-32C checksums using the `crc` library. It serves as the golden reference for all Verilog simulations and on-board validation. Run with `python CRC32_Verification.py` to execute built-in test vectors, or pass arguments for custom inputs.

### Parallel 32bit
A 32-bit parallel CRC-32C core generated by an open-source online tool (`bues.ch/h/crcgen`). It provides a functional baseline for the project.
- **rtl/GenCRC32.v** – Pure combinational 32-bit parallel CRC-32C module.
- **tb/crc_tb.v** – Testbench that applies four test vectors and compares outputs against expected values.

### Flat parallel 256bit
A single-cycle, 256-bit parallel CRC-32C engine. This implementation failed timing closure at the target frequency, which motivated the pipelined architecture.
- **GenCRC32_256.m** – MATLAB script that constructs the GF(2) matrices **A** and **B** and automatically generates the Verilog module `crc32_parallel_256.v`.
- **constraints/** – Contains the XDC file for a 100 MHz clock constraint (used to demonstrate the timing violation).
- **rtl/** – Verilog files for the combinational 256-bit engine.
- **tb/** – Testbench adapted from the 32-bit version for 256-bit verification.

### Pipelined (4 stages) parallel 256bit
The final, timing-clean implementation. It uses four identical 64-bit parallel CRC cores in a 4-stage pipeline to achieve 200 MHz operation with a fixed 4-cycle latency.
- **Pipeline_try1.m** – MATLAB script (derived from `GenCRC32_256.m`) that generates the 64-bit parallel CRC core `crc32_parallel_64.v`.
- **constraints/** – XDC file with 200 MHz clock constraint and package pin assignments.
- **rtl/** – Complete Verilog hierarchy:
  - `top.v` – Top-level wrapper integrating the pipeline core, VIO, and ILA debug cores.
  - `crc32_parallel256_pipe.v` – 4-stage pipeline core that instantiates four 64-bit CRC modules.
  - `crc32_parallel_64.v` – Auto-generated 64-bit parallel CRC-32C core (instantiated four times).
  - Additional IP files for VIO and ILA (generated by Vivado).
- **tb/** – Testbench for the pipelined design, enabling behavioural simulation with continuous 256-bit input streams.

## Reproducing Results

1. **Python golden model** – Run `python CRC32_Verification.py` to verify expected CRC values for the predefined test vectors.
2. **32-bit core** – Add `Parallel 32bit/rtl/GenCRC32.v` and its testbench to a Vivado project; run behavioural simulation.
3. **256-bit flat engine** – Run `GenCRC32_256.m` in MATLAB to regenerate `crc32_parallel_256.v`. Add the RTL files and the 100 MHz constraint to Vivado; run implementation and inspect timing violations.
4. **4-stage pipeline** – Run `Pipeline_try1.m` to regenerate `crc32_parallel_64.v`. Use the RTL files from `Pipelined (4 stages) parallel 256bit/rtl/` together with the 200 MHz constraint. The top-level module includes VIO/ILA for on-board debugging; for a production design, remove the debug cores and expose the data interface directly.
